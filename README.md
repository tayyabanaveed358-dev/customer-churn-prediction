# 📊 Telecom Customer Churn Prediction Pipeline

An end-to-end Machine Learning pipeline built in Python to address a critical business challenge: **Customer Retention**. This project engineers a robust data preprocessing workflow and benchmarks classification algorithms to identify at-risk customers before they churn.

---

## 🚀 Model Performance Summary

A direct comparison of the executed baseline models using the exact metrics generated by the code output:

| Model | Accuracy | Precision (Class 1) | Recall (Class 1) |
| :--- | :---: | :---: | :---: |
| **Logistic Regression** 🏆 | **81.97%** | **68.00%** | **60.00%** |
| **k-Nearest Neighbors (k-NN)** | 77.08% | 57.00% | 52.00% |

### 🔍 Key Takeaways
* **Logistic Regression** achieved the highest overall performance with **81.97% Accuracy**.
* In terms of catching actual churning customers (Class 1 Recall), **Logistic Regression** correctly identified **60%** of churn cases, outperforming k-NN by 8%.

## 🛠️ Tech Stack & Features
* **Language:** Python
* **Environment:** Jupyter Notebooks / VS Code
* **Libraries:** Pandas, NumPy, Scikit-Learn, Seaborn, Matplotlib, Kagglehub
* **Data Pipeline Features:**
  * Automated data retrieval using `kagglehub` API.
  * Robust data cleaning (handling invalid types and missing value imputation via medians).
  * Feature encoding using categorical dummy variables.
  * Standard Feature Scaling to guarantee distance-based algorithm performance.
